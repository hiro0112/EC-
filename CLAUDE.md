## プロジェクト概要

**EC-app** — 複数ECチャネル（楽天・Amazon・Shopify・Q10等）の売上・在庫を可視化するブラウザ完結型のダッシュボード。  
技術スタック: HTML / CSS / JavaScript（バンドラー不使用のバニラ構成）

**プライバシー方針**: すべてのデータ処理はクライアントサイドのみ。外部APIへのデータ送信は一切なし。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | UIマークアップ。タブ構成（①全商品売上 / ②商品別売上 / ③在庫分析） |
| `style.css` | CSS変数ベースのスタイル。`--primary` 等でテーマ管理 |
| `app.js` | CSV読み込み・解析・集計・Chart.js描画をすべて担う |

Chart.js 4.4 を jsdelivr CDN から読み込む（`index.html` 末尾）。

## 開発・実行

ビルド不要。ローカルサーバー経由でブラウザ確認する（ファイルプロトコルだと CORS 警告が出る場合がある）。

```bash
python -m http.server 8000   # http://localhost:8000
# または
npx serve .
```

## データ仕様（ネクストエンジン出力CSV）

エンコーディング: **Shift-JIS**。`TextDecoder('shift-jis')` で読み込む。

| 列インデックス (0始まり) | 内容 |
|---|---|
| 0 | 商品番号（10桁数字。それ以外はギフト等の特殊行として除外） |
| 1 | 商品名（SKU名。サイズ+カラーを含む長い文字列） |
| 2 | 売値 |
| 4 | 在庫数（当該時点） |
| 8 | 売上数量（対象期間内） |
| 9 | 売上金額（対象期間内） |

ファイル名から **チャネル** と **期間** を自動抽出する。  
例: `楽天・26年５月.csv` → `channel:"楽天"`, `period:"26年５月"`  
区切り文字: `・` `　` `_` `-`

## コアロジック（app.js）

### 商品グルーピング（`groupName` 関数）
同一商品が SKU（サイズ・カラー）ごとに別行になるため、集計時にまとめる。

- アンダースコア区切りの商品名 (`商品名_カラー_サイズ`) → 末尾2セグメントを除去
- スペース区切りの商品名 → 末尾2トークン（サイズ＋カラー）を除去

### タブ別データフロー

| タブ | 集計単位 | ソート |
|---|---|---|
| ①全商品売上 | チャネル別合計 | — |
| ②商品別売上 | グループ名（商品）別 | 降順（売上金額 or 数量） |
| ③在庫分析 | SKUコード別 | 残り月数 昇順（最短優先） |

### 在庫残月数の計算
```
月間販売数 = 全期間売上数 ÷ 読み込み期間数
残り月数   = 在庫数 ÷ 月間販売数
```
在庫あり・販売実績なし → ∞ として末尾に表示。

## コーディング規約

- `state` オブジェクト1つで UI 全体の状態を管理する
- Chart インスタンスは `state.charts` にキャッシュし、再描画前に `destroy()` する
- HTML を文字列で生成する際は `escHtml()` でサニタイズする
- CSS は `:root` の CSS 変数 (`--primary` 等) を通じて色を管理し、直書きしない
